# 网页版植物大战僵尸 v1.0 设计文档

**日期**：2026-04-30
**版本**：v1.0（基础完整版，单关卡）
**目标**：纯前端、无外部素材、桌面 + 移动响应式、可直接打开 `index.html` 游玩的高质量植物大战僵尸 web 游戏。

---

## 1. 范围与目标

本版本（v1.0）交付一个**单关卡完整体验**，作为后续扩展（多关卡、夜晚地图、更多植物/僵尸）的基础框架。

**完成标准**：
- 用户双击 `index.html` 即可在桌面浏览器游玩
- 手机横屏可玩（触屏操作）
- 一个完整关卡：3 波僵尸、6 种植物、3 种僵尸、约 3.5 分钟流程
- 有胜利和失败结算
- 有完整音效反馈
- 模块化代码结构，方便后续添加关卡/植物/僵尸

**非目标**（本版本不做，后续版本扩展）：
- 多关卡选择界面
- 进度保存
- 夜晚地图、泳池地图、屋顶地图
- 种子选择界面
- 排行榜、成就

## 2. 核心机制

### 2.1 网格

- 5 行 × 9 列
- 内部画布坐标：1000 × 600（CSS 等比缩放适配窗口）
- 单格尺寸：80 × 96 像素
- 草坪起始 x = 70，起始 y = 90
- 列 x 中心 = 110 + col × 80
- 行 y 中心 = 138 + row × 96

### 2.2 阳光经济

| 项 | 数值 |
|---|---|
| 起始阳光 | 50 |
| 天降阳光间隔 | 10 秒 |
| 天降阳光数量 | 25 |
| 天降阳光存活时长 | 10 秒（未点击则消失） |
| 向日葵产阳光间隔 | 24 秒 |
| 向日葵产阳光数量 | 25 |

### 2.3 胜负条件

- **胜利**：第 3 波僵尸已全部出场完毕，且场上无任何僵尸持续 3 秒
- **失败**：任一僵尸越过 x = 50 的左边界，且该行小推车已使用（或被消耗）
- **小推车**：每行最左侧 1 台，僵尸进入推车列（x ≤ 80）时启动，沿该行向右加速冲过，碾死沿途所有僵尸后消失（每行限用一次）

## 3. 植物（6 种）

| 名称 | 阳光成本 | 卡片冷却 | HP | 行为 |
|---|---|---|---|---|
| 🌻 向日葵 | 50 | 7.5s | 300 | 每 24 秒生成 1 个 25 阳光（落在自身位置） |
| 🌱 豌豆射手 | 100 | 7.5s | 300 | 同行右侧出现僵尸时，每 1.4 秒射出 1 颗豌豆（伤害 20） |
| 🌰 坚果墙 | 50 | 30s | 4000 | 不攻击；血量分 3 阶段视觉变化：满 / ≤2/3 出现裂纹 / ≤1/3 大裂纹 |
| 🍒 樱桃炸弹 | 150 | 50s | ∞ | 放下后 1.2 秒爆炸，3×3 网格范围内全部僵尸瞬间死亡，自身消失 |
| ❄️ 寒冰射手 | 175 | 7.5s | 300 | 同豌豆射手，伤害 20，附加减速效果（速度 ×0.5 持续 4 秒） |
| 🥔 土豆雷 | 25 | 30s | ∞ | 放下后 14 秒武装期（埋地，不攻击）；武装后僵尸踩中触发爆炸，3×3 网格范围秒杀 |

### 3.1 共同规则

- 同一格只能种 1 株植物（已占格不可重复种植，即使该植物正被啃咬）
- 选中卡片后跟随鼠标显示半透明预览，可种位置高亮（绿色），不可种红色
- 阳光不足或冷却中的卡片置灰，不可点击

### 3.2 攻击判定

- 豌豆射手 / 寒冰射手：扫描自己所在行，发现 x > 自己且未死的僵尸 → 进入射击循环
- 子弹：每帧 x += 速度 × dt（速度 350 px/s），出屏幕右边界即销毁；与同行僵尸 AABB 碰撞即命中
- 减速：寒冰豌豆命中后给僵尸加 4 秒减速 buff；多次命中刷新时长不叠加倍率

## 4. 僵尸（3 种）

| 名称 | HP | 速度 | 啃咬伤害 | 启示 |
|---|---|---|---|---|
| 普通僵尸 | 200 | 0.4 格/秒 ≈ 32 px/s | 100/秒 | — |
| 路障僵尸 | 200 + 360 路障 = 560 | 0.4 格/秒 | 100/秒 | 路障损坏分 3 阶段视觉变化 |
| 铁桶僵尸 | 200 + 1100 铁桶 = 1300 | 0.4 格/秒 | 100/秒 | 铁桶损坏分 3 阶段视觉变化 |

### 4.1 行为状态机

- `walking`：向左移动（gridX -= speed × dt × slowFactor）
- `eating`：前方 1 格内有植物 → 停止移动，每秒造成 100 伤害（每 0.5 秒触发一次咬一次伤害 50）
- `dying`：HP ≤ 0 → 翻倒动画 0.6 秒后销毁
- `slowed`：附加状态，速度 ×0.5，啃咬频率不变

### 4.2 入场

- 从画布右侧 x = 1010 出场
- 出场时播放出场音效 + 短暂从透明渐入

## 5. 关卡（Level 1）

### 5.1 流程

| 时间 | 事件 |
|---|---|
| 0–10 s | 教程提示："收集阳光，种下向日葵积累更多阳光" |
| 10 s | 第 1 波开始 |
| 10–45 s | 5 个普通僵尸，间隔 6 秒入场，行随机 |
| 45 s | 第 2 波开始：屏幕中央显示 "一大波僵尸即将来袭！"（1.5 秒动画） |
| 45–95 s | 10 只僵尸：7 普通 + 3 路障，间隔 4–5 秒，行随机 |
| 95 s | 第 3 波开始：再次显示 "最终大波僵尸！" |
| 95–180 s | 15 只僵尸：5 普通 + 6 路障 + 4 铁桶，间隔 3–4 秒，行随机 |
| 全部消灭 | 等待 3 秒确认场上无僵尸 → 显示 "胜利！" 画面 |

### 5.2 配置数据结构

`levels.js` 导出 `level1` 对象，含：
- `id`、`name`
- `availablePlants`：植物 ID 数组（v1.0 全部 6 种）
- `startingSun`：50
- `sunDropInterval`：10000 ms
- `waves`：数组，每项含 `triggerTime`、`zombies`（[{type, delay, row?}]）、`announceText`

后续加关卡只需在此追加 `level2`、`level3`...

## 6. 技术架构

### 6.1 渲染

- HTML5 Canvas 2D 单画布
- 内部坐标 1000×600；CSS 通过 `transform: scale()` 等比缩放铺满窗口
- 适配：监听 `resize` 事件重算缩放

### 6.2 主循环

```
requestAnimationFrame → loop(now):
    accumulator += now - lastTime
    while accumulator >= TIMESTEP:        // TIMESTEP = 1000/60
        update(TIMESTEP / 1000)
        accumulator -= TIMESTEP
    render()
```

固定步长保证不同设备游戏速度一致；render 自由帧率。

### 6.3 实体系统

每种实体统一接口：

```js
{
  update(dt, game),   // 推进状态
  render(ctx),        // 绘制
  isDead              // boolean，true 时被 game 在帧末过滤
}
```

`game` 维护分类数组：`plants`、`zombies`、`projectiles`、`suns`、`particles`、`mowers`。

### 6.4 坐标

- 僵尸用浮点 `gridX`（连续位移）+ 整数 `row`
- 植物用整数 `col` + `row`
- 子弹用浮点 `x` + 整数 `row`
- 像素坐标转换由 `grid.js` 提供工具函数

### 6.5 碰撞

- 子弹 vs 僵尸：同 row + 子弹 x ≥ 僵尸前缘（gridX 转 px - 20）→ 命中
- 僵尸 vs 植物：同 row + 同 col + 僵尸前缘进入植物中心 ±20 px → 啃咬
- 爆炸 vs 僵尸：在爆炸中心的 row±1 + col±1 范围内 → 死亡
- 小推车 vs 僵尸：同 row + 僵尸 x 位于推车 x ±30 → 死亡

### 6.6 文件结构

```
game-1/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── main.js          # 入口，启动游戏循环
│   ├── config.js        # 全部数值常量
│   ├── game.js          # GameState 类，波次、阳光、胜负
│   ├── grid.js          # 坐标转换、占用查询
│   ├── plants.js        # 6 种植物定义和行为
│   ├── zombies.js       # 3 种僵尸定义和行为
│   ├── projectiles.js   # 豌豆、寒冰豌豆
│   ├── sun.js           # 阳光实体
│   ├── particles.js     # 粒子效果工厂
│   ├── mower.js         # 小推车
│   ├── renderer.js      # Canvas 绘制函数集合
│   ├── ui.js            # 卡片栏、阳光计数、横幅、胜负画面
│   ├── input.js         # 鼠标 + 触屏处理
│   ├── audio.js         # Web Audio 音效合成
│   └── levels.js        # 关卡配置（v1.0 仅 level1）
└── docs/
    └── superpowers/
        └── specs/
            └── 2026-04-30-pvz-game-design.md  (本文档)
```

### 6.7 模块通信

- 全部使用 ES Modules（`<script type="module" src="js/main.js">`）
- 通过显式 import 而非全局变量
- `game` 单例由 `main.js` 创建，作为参数注入需要的模块（避免循环引用）

### 7.2 配色

- 草坪条纹：浅绿 `#9ec648` / 深绿 `#8ab83a` 交替
- 房子墙体：奶油色 `#f5e1a4`，屋顶深棕 `#5d3a1a`
- 出怪区土壤：`#6b4226`
- UI 木纹底色：`#8b5a2b`，亮色 `#d4a574`
- 阳光金色：`#ffcc33`
- 危险红：`#c44`
- 寒冰蓝：`#9fd3e8`

### 7.3 植物 Canvas 绘制

每株植物由代码绘制，不依赖图片。

- **🌻 向日葵**：黄色花瓣（12 片，长椭圆围成圆环）+ 棕色花心 + 绿色茎；产阳光时花心放大 1.2× 脉动 0.4 秒
- **🌱 豌豆射手**：圆形绿头 + 黑色椭圆嘴（默认闭合一条线）+ 茎；射击时嘴张开椭圆 0.15 秒后闭合
- **🌰 坚果墙**：棕色椭圆 + 两个黑点眼睛；血量阶段切换裂纹纹理（直线条）
- **🍒 樱桃炸弹**：两颗红色圆球 + 共用绿色茎 + 引信；引信发出火花粒子（黄白小点向上飘）；爆炸瞬间画白光圆环（扩张）+ 橙红填充
- **❄️ 寒冰射手**：蓝白渐变头 + 雪花样叶片；豌豆带浅蓝拖尾
- **🥔 土豆雷**：埋阶段只露顶部 + 文字 "准备中..."；武装后露出红色按钮 + 偶尔闪烁

### 7.4 僵尸 Canvas 绘制

- **普通僵尸**：灰绿身体（高 80px）+ 撕烂衣领（深灰锯齿）+ 头微歪 + 一只手前伸；走路：用 `sin(time × π × 2)` 控制左右腿前后摇摆
- **路障僵尸**：在普通僵尸头顶画橙色圆锥（可三段视觉：完整 / 缺角 / 残破）
- **铁桶僵尸**：在普通僵尸头顶画灰色梯形铁桶（三段：完整 / 凹陷 / 锈蚀）

### 7.5 动画与特效

| 事件 | 动画 |
|---|---|
| 植物种植 | 30% 缩放 + 透明，0.3s 缓动到 100% |
| 阳光下落 | 从屏幕上方掉落（重力 + 终点 y 锁定）；落定后旋转脉动 |
| 阳光收集 | 朝阳光计数器位置贝塞尔飞行 + 缩小，0.4s 到达消失 |
| 豌豆飞行 | 白色 4 像素拖尾；命中时 6 个黄绿粒子四散 |
| 寒冰命中 | 白色雪花粒子 + 僵尸覆盖半透明蓝色滤镜 4 秒 |
| 樱桃/土豆雷爆炸 | 0.05s 内白色冲击波从 0 扩到 1.5 格 + 橙红填充 + 烟雾粒子 |
| 僵尸死亡 | 身体翻倒（旋转 90°）+ 灵魂飘出（半透明白色椭圆向上漂浮 1s 渐隐） |
| 啃咬 | 植物轻微抖动（x ± 1px 高频）+ 上方浮动 "啊" 文字（0.3s 渐隐） |

### 7.6 UI 元素

- **植物卡片**（150×60，6 张横排底部居中）：木质背景 + 植物迷你图（30×30）+ 阳光成本数字；冷却时灰色蒙版从下到上消退；阳光不足时整张卡变暗 50%；选中时金色 3px 边框 + 光晕
- **阳光计数器**：左上角 80×40 金色徽章 + 数字
- **波次进度条**：右上角 3 个圆点，已完成绿色，当前闪烁，未来灰色
- **暂停按钮**：右上角，按下则游戏循环跳过 update，画布灰度化 + 中央显示 "暂停"
- **教程提示**：第一次开局显示 0–10s，半透明黑底白字浮在草坪上方
- **大波横幅**：占满屏幕宽度，半透明红黑底 + 大字 "一大波僵尸即将来袭！"
- **胜利画面**：金色大字 "胜利！" + 一袋金币掉落（粒子）+ "再玩一次" 按钮
- **失败画面**：暗红蒙版 + "僵尸吃掉了你的脑子！" + "重新开始" 按钮

### 7.7 响应式

- 桌面（≥800px）：画布按等比缩放，最大宽度 1000px，垂直居中
- 平板/手机（<800px）：检测设备方向，竖屏时显示半透明蒙版 "请将设备横屏"
- 横屏后：画布等比铺满（保持 5:3 比例）
- 触屏：tap = 鼠标点击；长按 0.5 秒已种植物 → 弹出"铲除"按钮

## 8. 音效（Web Audio 合成）

| # | 事件 | 合成 |
|---|---|---|
| 1 | 阳光收集 | 1200 Hz 正弦波，包络 attack 0.01s / decay 0.15s |
| 2 | 种植植物 | 200 Hz 三角波，0.1s 衰减 |
| 3 | 豌豆射出 | 300→600 Hz 正弦滑音，0.08s |
| 4 | 豌豆命中 | 白噪声 0.05s + 低通滤波 |
| 5 | 寒冰命中 | 2000 Hz 正弦 0.15s 衰减 |
| 6 | 爆炸 | 白噪声 0.4s + 包络 + 低通滤波下降 |
| 7 | 僵尸啃咬 | 200 Hz 方波 50ms 短促重复 2 次 |
| 8 | 僵尸出场嗥叫 | 100 Hz 锯齿波，0.6s 慢衰减 |
| 9 | 小推车启动 | 锯齿波 + 频率 100→400Hz 0.3s 上升 |
| 10 | 大波警报 | 800/400 Hz 高低音交替方波 0.4s |

`audio.js` 暴露 `play(name)` 函数；首次用户交互后初始化 `AudioContext`（解决浏览器自动播放限制）。

## 9. 输入处理

### 9.1 桌面

- `mousemove`：更新选中植物预览位置 + 高亮可种格
- `mousedown` + `mouseup`：
  - 植物卡片栏 → 选中/取消选中
  - 卡片栏外 + 已选中植物 + 落点为空格且阳光足 → 种植 + 扣阳光 + 启动卡片冷却
  - 卡片栏外 + 未选中 + 落点为阳光 → 收集
  - 落点为已种植物 → 暂无操作（v1.0 不做铲除）
- `Esc`：取消选中

### 9.2 触屏

- `touchstart` 等同 `mousedown`
- 长按 0.5s 在已种植物上 → 弹出铲除确认按钮

## 10. 关键数据流

### 10.1 一帧的处理顺序

1. `input.processQueue()` — 处理本帧累积的输入事件
2. `game.spawnSchedule()` — 检查是否到了出怪/出阳光时机
3. 各实体 `update(dt, game)`：
    - 阳光：下落 / 旋转
    - 植物：找目标 / 攻击冷却 / 产阳光
    - 子弹：移动 / 命中检测
    - 僵尸：移动 / 啃咬 / 受击
    - 粒子：寿命衰减
    - 小推车：触发 / 移动 / 碰撞
4. `game.checkCollisions()` — 子弹↔僵尸、爆炸↔僵尸、推车↔僵尸
5. `game.cullDead()` — 移除 isDead 实体
6. `game.checkWinLose()` — 判断胜负
7. `renderer.render(ctx, game)` — 顺序绘制：背景 → 网格 → 植物 → 阳光 → 子弹 → 僵尸 → 粒子 → UI

### 10.2 种植事件

```
input → ui.onCardClick(plantId)
  → game.selectPlant(plantId) (检查阳光/冷却)
input → ui.onCanvasClick(x, y)
  → if game.selectedPlant && grid.isEmpty(col, row):
       game.placePlant(plantId, col, row)
       game.spendSun(cost)
       game.startCardCooldown(plantId)
       audio.play('plant')
       particles.spawnPlantingPoof(col, row)
```

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| Canvas 绘制角色不够精致 | 视觉品质 | 多用渐变、阴影、动画补偿；每个角色绘制函数允许后续单独优化 |
| 移动端性能 | 卡顿 | 限制最多同屏粒子 200 个；scaleFactor 可降至 0.6 应急 |
| Web Audio 浏览器自动播放限制 | 没声音 | 首次点击/触摸后才创建 AudioContext，UI 上无需用户感知 |
| 数值平衡难调 | 关卡过难/过易 | 全部数值集中在 `config.js`，方便快速调参 |
| 铲除功能 v1.0 不做 | 玩家误种无法挽回 | 选中植物时显示半透明预览 + 高亮可种格；按 Esc 取消 |
